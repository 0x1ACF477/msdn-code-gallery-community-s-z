Introduction

Sometimes some people ask me about the use of Entity Framework, in main the Code First, with Sql Server Compact Edition.

You can to read my blog post about it here.

The Sql Compact Edition

This version is the simpler of Sql Server, here your data is saved in one file .sdf, and not a file .mdf that needs of Sql Server run in the machine

Is a version for apps of the old Windows Mobile or another little application tha don’t need of the all resources of the Sql Server. Is a embedded version: you just need add the file.

Using the Entity Framework with SQL CE

The EF in your 4.0 version support work with SQL CE in the model first, database first and tje code first. The way for model and database first are simple:

  • Add the edmx in your project
  • Select the connection with your database. Is possible that here you don’t find the connector for Sql Ce 4.0, so you can to select the version 3.5
  • After you need go to your web.config
  • In the connection string of your emdx change the propriety ProviderManifestTokenfrom 3.5 to 4.0

that's all!
